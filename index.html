Q 1.1 Explain the Structure of a C++ Program with a suitable example. 13 Marks
C++ is an object-oriented programming language developed by Bjarne Stroustrup as an extension of C. A C++ program is organized into several logical sections that work together to produce output.

1. Basic Structure of a C++ Program

Every C++ program consists of the following sections:

  • Documentation Section: Contains comments (// or /* */) describing the program purpose, author, and date.
  • Preprocessor Directives: Lines beginning with # that instruct the preprocessor (e.g., #include <iostream>).
  • Namespace Declaration: using namespace std; allows use of standard library functions without the std:: prefix.
  • Global Declarations: Variables, constants, or function prototypes declared outside all functions; accessible throughout the program.
  • Class Declaration: Blueprint for creating objects (used in OOP).
  • Function Definitions: Blocks of code that perform specific tasks.
  • main() Function: The entry point of every C++ program; execution begins here.

2. Detailed Explanation of Each Section

  • Header Files (#include): Provide declarations for built-in functions. <iostream> provides cin/cout; <cmath> provides math functions.
  • main() Return Type: Returns int to the OS (0 = success, non-zero = error).
  • Statements & Expressions: Instructions ending with semicolon (;) that the computer executes sequentially.
  • Return Statement: return 0; signals successful completion.

3. Example Program

// Documentation Section
// Program: Simple C++ Structure Demo
// Author: Student | Date: 2024

#include <iostream>   // Preprocessor Directive
#include <string>

using namespace std;   // Namespace Declaration

int marks = 0;         // Global Declaration

// Function Prototype
void displayInfo(string name, int age);

// Class Declaration
class Student {
public:
    string name;
    int roll;
    void display() {
        cout << "Roll: " << roll << " Name: " << name << endl;
    }
};

// Function Definition
void displayInfo(string name, int age) {
    cout << "Name: " << name << " Age: " << age << endl;
}

// main() Function – Entry Point
int main() {
    Student s1;
    s1.roll = 101;
    s1.name = "Raj";
    s1.display();
    displayInfo("Priya", 20);
    cout << "Program executed successfully" << endl;
    return 0;
}

4. Output

Roll: 101 Name: Raj
Name: Priya Age: 20
Program executed successfully

5. Key Points

  • Every C++ program must have exactly one main() function.
  • Statements end with a semicolon (;).
  • C++ is case-sensitive: Main ≠ main.
  • Curly braces { } define the body of functions and classes.

Q 1.3 Explain Control Structures in C++ with examples (if, if-else, switch, loops). 13 Marks
Control structures determine the flow of execution in a program. C++ supports three categories: Sequential, Selection (Branching), and Iteration (Looping).

1. Selection Statements

a) if statement:

if (condition) {
    // executes when condition is true
}

b) if-else statement:

if (marks >= 50)
    cout << "Pass";
else
    cout << "Fail";

c) if-else if ladder:

if (marks >= 90)       cout << "Grade A";
else if (marks >= 75) cout << "Grade B";
else if (marks >= 60) cout << "Grade C";
else                  cout << "Grade D";

d) switch statement:

switch (day) {
    case 1: cout << "Monday";  break;
    case 2: cout << "Tuesday"; break;
    default: cout << "Other";
}

2. Iteration Statements (Loops)

Q 1.4 Explain Function Overloading in C++ with a complete example. 13 Marks
Function Overloading is the ability to define multiple functions with the same name but different parameter lists (different number, type, or order of parameters). The compiler distinguishes them at compile time — a form of compile-time (static) polymorphism.

1. Rules for Function Overloading

  • Functions must differ in the number of parameters, or
  • Functions must differ in the type of parameters, or
  • Functions must differ in the order of parameters.
  • Return type alone cannot distinguish overloaded functions.

2. Complete Example

#include <iostream>
using namespace std;

// Overloaded add() functions
int    add(int a, int b)              { return a + b; }
double add(double a, double b)        { return a + b; }
int    add(int a, int b, int c)      { return a + b + c; }
string add(string a, string b)       { return a + b; }

int main() {
    cout << add(5, 3)             << endl; // calls int version → 8
    cout << add(3.5, 2.1)         << endl; // calls double version → 5.6
    cout << add(1, 2, 3)           << endl; // calls 3-param version → 6
    cout << add("Hello, ", "World") << endl; // string concat
    return 0;
}

3. Overloading with Default Arguments

void display(string name, int age = 18, string city = "Chennai") {
    cout << name << " | Age: " << age << " | " << city << endl;
}
// Calls: display("Ram"); display("Sam",20); display("Raj",21,"Madurai");

4. Advantages

  • Improves code readability — same logical name for similar operations.
  • Supports compile-time polymorphism.
  • Reduces need to invent different function names like addInt, addDouble.
  • Makes programs more intuitive and maintainable.

Q 2.1 Explain Class and Object in C++ with programs demonstrating encapsulation. 13 Marks
A Class is a user-defined data type that acts as a blueprint for objects. It combines data (attributes) and functions (methods) into a single unit — implementing encapsulation. An Object is a runtime instance of a class.

1. Class Declaration Syntax

class ClassName {
private:
    // data members (hidden from outside)
public:
    // member functions (interface)
protected:
    // accessible to derived classes
};

2. Access Specifiers

SpecifierAccessible From
privateOnly within the same class
publicAnywhere in the program
protectedSame class + derived classes

3. Complete Program – Bank Account Class

#include <iostream>
#include <string>
using namespace std;

class BankAccount {
private:
    string owner;
    double balance;          // encapsulated data

public:
    // Constructor
    BankAccount(string name, double bal) {
        owner   = name;
        balance = bal;
    }

    void deposit(double amt) {
        if(amt > 0) balance += amt;
        cout << "Deposited: " << amt << endl;
    }

    void withdraw(double amt) {
        if(amt <= balance) { balance -= amt; cout << "Withdrawn: " << amt << endl; }
        else cout << "Insufficient balance!" << endl;
    }

    void display() {
        cout << "Owner: " << owner << " | Balance: Rs." << balance << endl;
    }
};

int main() {
    BankAccount acc("Karthik", 5000.0);
    acc.display();
    acc.deposit(2000);
    acc.withdraw(1500);
    acc.display();
    return 0;
}

4. Key Concepts

  • Encapsulation: Data and methods are bundled; private data accessed only through public methods (getters/setters).
  • Data Hiding: Internal state is protected from direct external modification.
  • Member Functions: Can be defined inside the class (inline) or outside using the scope resolution operator (::).
  • Object Creation: ClassName obj; (stack) or ClassName *obj = new ClassName(); (heap).
Q 2.2 Explain Static Data Members and Static Member Functions in C++ with example. 13 Marks
A static data member is shared among all objects of a class — only one copy exists in memory regardless of how many objects are created. A static member function can access only static data members.

1. Static Data Members

  • Declared with static keyword inside the class.
  • Defined and initialized outside the class.
  • Shared by all objects — acts like a class-level variable.
  • Accessed as ClassName::member or via any object.

2. Static Member Functions

  • Declared with static keyword.
  • Can be called without creating an object: ClassName::func().
  • Can access only static data members; no this pointer.
  • Cannot access non-static members directly.

3. Complete Example – Student Counter

#include <iostream>
using namespace std;

class Student {
private:
    string name;
    int id;

public:
    static int count;    // static data member

    Student(string n, int i) : name(n), id(i) {
        count++;          // incremented for each object
    }

    static void showCount() {   // static member function
        cout << "Total students: " << count << endl;
    }

    void display() {
        cout << "ID: " << id << " Name: " << name << endl;
    }
};

// Static member defined outside
int Student::count = 0;

int main() {
    Student::showCount();   // 0 – no object yet

    Student s1("Arun", 1);
    Student s2("Bala", 2);
    Student s3("Chitra", 3);

    Student::showCount();   // 3
    s1.display(); s2.display(); s3.display();
    return 0;
}

4. Output

Total students: 0
Total students: 3
ID: 1 Name: Arun
ID: 2 Name: Bala
ID: 3 Name: Chitra

Q 2.3 Explain the Types of Constructors in C++ with programs. 13 Marks
A Constructor is a special member function that is automatically invoked when an object is created. It has the same name as the class and no return type. It is used to initialize the object's data members.

1. Types of Constructors

a) Default Constructor — No parameters; compiler provides one if not defined.

class Box {
public:
    int length, width;
    Box() { length = width = 0; cout << "Default Constructor\n"; }
};

b) Parameterized Constructor — Accepts arguments to initialize members.

class Box {
public:
    int l, w, h;
    Box(int a, int b, int c) : l(a), w(b), h(c) {}
    int volume() { return l * w * h; }
};

c) Copy Constructor — Initializes an object from another object of the same class.

class Box {
public:
    int l, w;
    Box(int a, int b) { l=a; w=b; }
    Box(const Box &obj) {    // Copy Constructor
        l = obj.l; w = obj.w;
        cout << "Copy Constructor invoked\n";
    }
};
// Usage: Box b1(10,5); Box b2(b1);  ← copy constructor called

d) Constructor Overloading — Multiple constructors with different parameters.

#include <iostream>
using namespace std;
class Rectangle {
    int l, w;
public:
    Rectangle()            { l = w = 0; }
    Rectangle(int s)        { l = w = s; }         // square
    Rectangle(int a,int b) { l = a; w = b; }
    void area() { cout << "Area = " << l*w << endl; }
};
int main() {
    Rectangle r1;           r1.area();  // 0
    Rectangle r2(5);        r2.area();  // 25
    Rectangle r3(4,7);      r3.area();  // 28
    return 0;
}

2. Destructor

  • Special function called when object goes out of scope or is deleted.
  • Syntax: ~ClassName() { }
  • Used to release dynamically allocated memory.
  • Only one destructor per class; cannot be overloaded.

Q 2.4 Explain Object Arrays and Pointers in C++ with programs. 13 Marks
An array of objects stores multiple objects of the same class in contiguous memory. A pointer to an object stores the address of an object and allows dynamic memory management.

1. Array of Objects

#include <iostream>
using namespace std;
class Student {
public:
    string name; int marks;
    void input() { cin >> name >> marks; }
    void display() { cout << name << ": " << marks << endl; }
};
int main() {
    Student s[3];          // array of 3 Student objects
    for(int i=0; i<3; i++) s[i].input();
    for(int i=0; i<3; i++) s[i].display();
    return 0;
}

2. Pointers to Objects

class Point {
public:
    int x, y;
    Point(int a, int b): x(a), y(b){}
    void show() { cout << "("<<x<<","<<y<<")"<<endl; }
};
int main() {
    Point p1(3,4);
    Point *ptr = &p1;         // pointer to object
    ptr->show();              // access via arrow operator

    Point *dyn = new Point(10,20); // dynamic allocation
    dyn->show();
    delete dyn;               // release memory

    // Dynamic array of objects
    Point *arr = new Point[2]{{1,2},{3,4}};
    for(int i=0; i<2; i++) (arr+i)->show();
    delete[] arr;
    return 0;
}

3. this Pointer

  • this is an implicit pointer available in all non-static member functions.
  • It points to the object that invoked the member function.
  • Used to resolve naming conflicts and return the current object.
class MyClass {
    int x;
public:
    MyClass(int x) { this->x = x; }  // this->x = member, x = param
    MyClass& setX(int v) { x = v; return *this; } // method chaining
};

Q 3.1 Explain the Types of Inheritance in C++ with programs. 13 Marks
Inheritance is the mechanism by which a class (derived/child) acquires properties and behaviors of another class (base/parent). It promotes code reusability and establishes an IS-A relationship.

1. Single Inheritance

One derived class inherits from one base class.

class Animal {
public:
    void breathe() { cout << "Breathing...\n"; }
};
class Dog : public Animal {
public:
    void bark() { cout << "Barking...\n"; }
};
// Dog d; d.breathe(); d.bark();

2. Multiple Inheritance

One derived class inherits from multiple base classes.

class Father { public: void property() { cout << "Father's property\n"; }};
class Mother { public: void talent()   { cout << "Mother's talent\n";   }};
class Child : public Father, public Mother {
public:
    void show() { property(); talent(); }
};

3. Multilevel Inheritance

Inheritance in a chain: A → B → C.

class Vehicle  { public: void move()  { cout << "Moving\n";     }};
class Car     : public Vehicle {
public: void drive() { cout << "Driving\n"; }};
class SportsCar: public Car     {
public: void turbo() { cout << "Turbo boost!\n"; }};
// SportsCar sc; sc.move(); sc.drive(); sc.turbo();

4. Hierarchical Inheritance

Multiple derived classes from one base class.

class Shape  { public: void draw() { cout << "Drawing shape\n";  }};
class Circle: public Shape { public: void area() { cout << "Circle area\n"; }};
class Square: public Shape { public: void area() { cout << "Square area\n"; }};

5. Hybrid Inheritance

Combination of two or more types of inheritance. May lead to the Diamond Problem resolved using virtual base classes.

class A { public: int x; };
class B : virtual public A {};     // virtual to avoid duplicate
class C : virtual public A {};
class D : public B, public C {};   // D gets only one copy of A

Q 3.2 Explain Function Overriding in C++ with a complete program. 13 Marks
Function Overriding occurs when a derived class defines a function with the same name, same parameters, and same return type as a function in the base class. The derived class version replaces the base class version — enabling run-time polymorphism.

1. Key Characteristics

  • Requires inheritance — works only in derived classes.
  • Function signature must be identical (name + parameters + return type).
  • When called via base class pointer/reference with virtual, the derived version executes.
  • Without virtual, the base class version is called (static binding).

2. Complete Program

#include <iostream>
using namespace std;

class Shape {
public:
    virtual void draw() {           // virtual enables overriding
        cout << "Drawing a generic shape\n";
    }
    virtual double area() { return 0; }
};

class Circle : public Shape {
    double radius;
public:
    Circle(double r) : radius(r) {}
    void draw() override {          // overrides base version
        cout << "Drawing Circle with radius " << radius << endl;
    }
    double area() override { return 3.14 * radius * radius; }
};

class Rectangle : public Shape {
    double l, w;
public:
    Rectangle(double a,double b): l(a), w(b){}
    void draw() override {
        cout << "Drawing Rectangle " << l << "x" << w << endl;
    }
    double area() override { return l * w; }
};

int main() {
    Shape *s;                        // base class pointer

    Circle c(5.0);
    s = &c;
    s->draw();                       // Circle's draw called (runtime)
    cout << "Area: " << s->area() << endl;

    Rectangle r(4.0, 6.0);
    s = &r;
    s->draw();
    cout << "Area: " << s->area() << endl;
    return 0;
}

3. Overloading vs Overriding

ClassSame classDifferent classes (inheritance)
ParametersMust differMust be same
BindingCompile-timeRun-time (with virtual)
Keyword—virtual / override
Q 3.3 Explain Abstract Class in C++ with example programs. 13 Marks
An Abstract Class is a class that contains at least one pure virtual function. It cannot be instantiated (no objects can be created directly). It serves as a base class interface that derived classes must implement.

1. Pure Virtual Function

  • Declared with = 0 at the end: virtual void func() = 0;
  • Has no body in the abstract class.
  • All derived (concrete) classes must override it.

2. Complete Example

#include <iostream>
using namespace std;

class Employee {           // Abstract Class
protected:
    string name;
    int id;
public:
    Employee(string n, int i): name(n), id(i) {}
    virtual double calculateSalary() = 0;  // Pure virtual
    virtual void display() {
        cout << "ID: "<<id<<" Name: "<<name;
    }
    virtual ~Employee() {}
};

class FullTime : public Employee {
    double monthlySalary;
public:
    FullTime(string n, int i, double s)
        : Employee(n,i), monthlySalary(s) {}
    double calculateSalary() override { return monthlySalary; }
    void display() override {
        Employee::display();
        cout << " [FullTime] Salary: Rs." << calculateSalary() << endl;
    }
};

class PartTime : public Employee {
    int hours; double rate;
public:
    PartTime(string n, int i, int h, double r)
        : Employee(n,i), hours(h), rate(r){}
    double calculateSalary() override { return hours * rate; }
    void display() override {
        Employee::display();
        cout << " [PartTime] Salary: Rs." << calculateSalary() << endl;
    }
};

int main() {
    Employee *e[2];
    e[0] = new FullTime("Meena", 101, 35000);
    e[1] = new PartTime("Kumar", 102, 80, 150);
    for(int i=0; i<2; i++) e[i]->display();
    delete e[0]; delete e[1];
    return 0;
}

3. Rules and Key Points

  • Objects of abstract class cannot be created.
  • Pointers and references to abstract class can be created.
  • If a derived class does not override all pure virtual functions, it also becomes abstract.
  • Abstract classes define a common interface for all subclasses.
Q 3.4 Explain Virtual Functions and Runtime Polymorphism in C++. 13 Marks
A Virtual Function is a member function declared in the base class with the virtual keyword. When called through a base class pointer/reference, the correct derived class version is called at runtime — this is Runtime Polymorphism (Dynamic Dispatch).

1. How Virtual Functions Work – vTable

  • Compiler creates a virtual function table (vTable) for each class with virtual functions.
  • Each object contains a vPtr (pointer to its class's vTable).
  • At runtime, the correct function is found by looking up the vTable — hence dynamic binding.
vTable for Circle: [ draw → Circle::draw, area → Circle::area ]
vTable for Square: [ draw → Square::draw, area → Square::area ]

2. Complete Program

#include <iostream>
using namespace std;

class Animal {
public:
    virtual void speak() {       // virtual function
        cout << "Animal speaks\n";
    }
    virtual ~Animal() {}          // virtual destructor (best practice)
};

class Dog : public Animal {
public:
    void speak() override { cout << "Dog: Woof!\n";  }
};
class Cat : public Animal {
public:
    void speak() override { cout << "Cat: Meow!\n"; }
};
class Cow : public Animal {
public:
    void speak() override { cout << "Cow: Moo!\n";  }
};

void makeNoise(Animal *a) {    // polymorphic function
    a->speak();                   // resolved at runtime
}

int main() {
    Animal *animals[3] = { new Dog(), new Cat(), new Cow() };
    for(int i=0; i<3; i++) {
        makeNoise(animals[i]);
        delete animals[i];
    }
    return 0;
}

3. Virtual Destructor

  • When deleting a derived object through a base pointer, without virtual destructor only the base destructor is called → memory leak.
  • Always declare destructor virtual in base classes with virtual functions.

Q 4.1 Explain Operator Overloading (Binary Operator) in C++ with complete programs. 13 Marks
Operator Overloading allows C++ operators to be redefined to work with user-defined data types (classes). This enables operators like +, –, *, ==, << to work on objects — a form of compile-time polymorphism.

1. Syntax

return_type operator symbol (parameters) {
    // operator logic
}

2. Binary Operator Overloading – (+) for Complex Numbers

#include <iostream>
using namespace std;

class Complex {
private:
    double real, imag;
public:
    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    // Overload + operator
    Complex operator+(const Complex &c) {
        return Complex(real + c.real, imag + c.imag);
    }

    // Overload – operator
    Complex operator-(const Complex &c) {
        return Complex(real - c.real, imag - c.imag);
    }

    // Overload * operator
    Complex operator*(const Complex &c) {
        return Complex(real*c.real - imag*c.imag,
                        real*c.imag + imag*c.real);
    }

    // Overload == operator
    bool operator==(const Complex &c) {
        return (real == c.real && imag == c.imag);
    }

    // Overload << (friend function)
    friend ostream& operator<<(ostream &out, const Complex &c) {
        out << c.real;
        if(c.imag >= 0) out << "+";
        out << c.imag << "i";
        return out;
    }
};

int main() {
    Complex c1(3, 4), c2(1, -2);
    cout << "c1 = "     << c1 << endl;
    cout << "c2 = "     << c2 << endl;
    cout << "c1+c2 = "  << c1 + c2 << endl;
    cout << "c1-c2 = "  << c1 - c2 << endl;
    cout << "c1*c2 = "  << c1 * c2 << endl;
    cout << "Equal? "   << (c1 == c2 ? "Yes" : "No") << endl;
    return 0;
}

3. Operators That Cannot Be Overloaded

  • :: — Scope resolution
  • . — Member access
  • .* — Pointer-to-member
  • ?: — Ternary operator
  • sizeof, typeid

4. Rules for Operator Overloading

  • At least one operand must be a user-defined type.
  • Overloading cannot change operator precedence or associativity.
  • Overloading cannot change the number of operands.
  • Use friend functions when the left operand is not a class object (e.g., cout <<).
Q 4.2 Explain Function Templates in C++ with programs. 13 Marks
A Function Template is a blueprint for creating generic functions that can work with any data type. The actual data type is determined at compile time when the function is called — this is generic programming.

1. Syntax

template <typename T>
T functionName(T param1, T param2) {
    // function body
}

2. Complete Example – Generic Functions

#include <iostream>

// Template function – maximum
template <typename T>
T getMax(T a, T b) {
    return (a > b) ? a : b;
}

// Template with two type parameters
template <typename T1, typename T2>
void printPair(T1 x, T2 y) {
    cout << "(" << x << ", " << y << ")\n";
}

// Template – swap values
template <typename T>
void swapValues(T &a, T &b) {
    T temp = a; a = b; b = temp;
}

// Template – bubble sort
template <typename T>
void bubbleSort(T arr[], int n) {
    for(int i=0; i<n-1; i++)
        for(int j=0; j<n-i-1; j++)
            if(arr[j] > arr[j+1])
                swapValues(arr[j], arr[j+1]);
}

int main() {
    cout << getMax(10, 20)       << endl; // int → 20
    cout << getMax(3.14, 2.72)   << endl; // double → 3.14
    cout << getMax('z', 'a')      << endl; // char → z

    printPair(42, "hello");

    int arr[] = {5,2,8,1,9};
    bubbleSort(arr, 5);
    for(int x : arr) cout << x << " ";
    cout << endl;
    return 0;
}

Q 4.3 Explain Exception Handling in C++ with try, catch, and throw. 13 Marks
Exception Handling is a mechanism to handle runtime errors gracefully, preventing abrupt program termination. C++ uses three keywords: try (monitor code), throw (raise exception), and catch (handle exception).

#include <iostream>

Q 4.4 Explain Multiple Catch Blocks and Custom Exception Classes in C++. 13 Marks
Multiple catch blocks allow handling different types of exceptions separately. A custom exception class is a user-defined class derived from std::exception to represent application-specific errors.

1. Multiple Catch Blocks

#include <iostream>
#include <stdexcept>
using namespace std;

void process(int type) {
    switch(type) {
        case 1: throw 10;                              // int
        case 2: throw 3.14;                            // double
        case 3: throw string("String error");          // string
        case 4: throw out_of_range("Index exceeded");  // STL
    }
}

int main() {
    for(int t = 1; t <= 5; t++) {
        try {
            process(t);
        }
        catch (int e)            { cout << "Int exception: "    << e << endl; }
        catch (double e)         { cout << "Double exception: "  << e << endl; }
        catch (string e)         { cout << "String exception: "  << e << endl; }
        catch (out_of_range &e)  { cout << "Range exception: "   << e.what() << endl; }
        catch (...)              { cout << "Unknown exception!\n"; }
    }
    return 0;
}

2. Custom Exception Class

#include <iostream>
#include <exception>
using namespace std;

class InsufficientFundsException : public exception {
    double amount;
public:
    InsufficientFundsException(double a) : amount(a) {}
    const char* what() const noexcept override {
        return "Insufficient funds in account!";
    }
    double getAmount() { return amount; }
};

void withdraw(double balance, double amount) {
    if(amount > balance)
        throw InsufficientFundsException(amount - balance);
    cout << "Withdrawal successful: Rs." << amount << endl;
}

int main() {
    try {
        withdraw(5000, 3000);   // OK
        withdraw(5000, 7000);   // throws
    }
    catch (InsufficientFundsException &e) {
        cout << "Exception: " << e.what() << endl;
        cout << "Shortfall: Rs." << e.getAmount() << endl;
    }
    return 0;
}

3. Re-throwing Exceptions

catch(int e) {
    cout << "Caught in inner: " << e << endl;
    throw;  // re-throw to outer handler
}

4. Rules for Multiple Catch

  • Catch blocks are evaluated top to bottom — more specific before general.
  • catch(...) must always be the last catch block.
  • A base class catch can catch derived class exceptions.
  • Only one catch block executes per thrown exception.
